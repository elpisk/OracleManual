## 하(기초) 모범답안

| 번호 | 정답 | 해설 |
|------|------|------|
| 1 | ③ | Oracle 10g부터 POSIX 정규 표현식 지원 |
| 2 | ② | REGEXP_LIKE: WHERE 절에서 패턴 일치 여부 검사 (LIKE의 강력한 대체) |
| 3 | ② | `.`: 임의의 문자 하나 (개행 제외) |
| 4 | ③ | `^Oracle`: "Oracle"로 시작하는 문자열 |
| 5 | ② | `[[:digit:]]`: POSIX 숫자 클래스 = [0-9] |
| 6 | ② | `[aeiou]`: 문자 클래스 — a, e, i, o, u 중 하나에 일치 |
| 7 | ② | `+`: 앞 패턴 1회 이상 반복 (`*`은 0회 이상) |
| 8 | ③ | `{3,5}`: 3회 이상 5회 이하 반복 |
| 9 | ② | REGEXP_REPLACE: 패턴 일치 부분을 다른 문자열로 교체 |
| 10 | ③ | REGEXP_INSTR: 패턴이 발생하는 위치(숫자) 반환 |
| 11 | ① | REGEXP_SUBSTR: 패턴에 일치하는 부분 문자열 추출 |
| 12 | ③ | REGEXP_COUNT: Oracle 11g 이상에서 지원 |
| 13 | ③ | `$`: 문자열의 끝을 나타내는 앵커 |
| 14 | ② | `[^0-9]`: 부정 문자 클래스 — 숫자가 아닌 문자 |
| 15 | ② | `'i'`: case-insensitive (대소문자 구분 없음) |
| 16 | ② | `[[:alpha:]]`: POSIX 알파벳 클래스 = [a-zA-Z] |
| 17 | ② | `|`: OR 연산 (예: `cat|dog` → cat 또는 dog) |
| 18 | ③ | `?`: 앞 패턴 0회 또는 1회 (optional) |
| 19 | ② | `[^[:digit:]]`를 `''`로 교체 → 숫자 아닌 문자 모두 삭제, 숫자만 남음 |
| 20 | ② | CHECK 제약 + 정규 표현식 = 복잡한 형식 검증을 DB 수준에서 강제 |

## 중(응용) 모범답안

---

**[21번] 정답: ②**

`^[[:alpha:]]+$`: 처음부터 끝까지 알파벳만이어야 함. `'Hello123'`에 숫자 `123`이 있으므로 FALSE.

```sql
SELECT CASE WHEN REGEXP_LIKE('Hello123', '^[[:alpha:]]+$')
            THEN 'TRUE' ELSE 'FALSE' END AS result
FROM dual;
-- 결과: FALSE
```

---

**[22번] 정답: ②**

`^010-[0-9]{4}-[0-9]{4}$`:
- `^010-`: "010-"으로 시작
- `[0-9]{4}`: 숫자 정확히 4자리
- `-`: 구분자
- `[0-9]{4}`: 숫자 정확히 4자리
- `$`: 문자열 끝

①은 숫자가 1자리씩만 허용, ③은 `.`(임의 문자), ④는 `^`와 `$`가 없어 부분 일치.

---

**[23번] 정답: ②**

`' {2,}'`: 공백 2개 이상에 일치 → 하나의 공백으로 교체.
`'Hello    World'` → 공백 4개 → 1개로 → `'Hello World'`.

---

**[24번] 정답: ②**

`'abc123def'`에서 `[[:digit:]]`(숫자)가 처음 나타나는 위치: `a(1)b(2)c(3)1(4)` → 4.

---

**[25번] 정답: ②**

`'Apple,Banana,Cherry'`에서 `[^,]+`(쉼표가 아닌 문자의 연속):
- 1번째: Apple
- 2번째: Banana
- 3번째: Cherry
`occurrence=2` → Banana

---

**[26번] 정답: ③**

`'aababab'`에서 `'ab'` 발생:
- 위치 2-3: `ab`
- 위치 4-5: `ab`
- 위치 6-7: `ab`
총 3회.

---

**[27번] 정답: ②**

서브표현식 역참조:
- `([0-9]{2})` → `\1` = MM
- `([0-9]{2})` → `\2` = DD
- `([0-9]{4})` → `\3` = YYYY
- `'\3-\1-\2'` → YYYY-MM-DD

```sql
SELECT REGEXP_REPLACE('12/25/2024',
       '([0-9]{2})/([0-9]{2})/([0-9]{4})',
       '\3-\1-\2') FROM dual;
-- 결과: 2024-12-25
```

---

**[28번] 정답: ②**

`'^[[:lower:]]+$'`: 처음(`^`)부터 끝(`$`)까지 소문자(`[[:lower:]]`)만 1자 이상(`+`).
①은 소문자가 하나라도 있으면 일치 (대문자 포함 가능), ③은 대문자 없음이지만 공백 등 허용.

---

**[29번] 정답: ②**

`'010-1234-5678'`에서 `[0-9]+` 발생:
- 1번째: 010
- 2번째: 1234
`occurrence=2` → 1234

---

**[30번] 정답: ③**

`^[[:upper:]][[:lower:]]+$`:
- `^[[:upper:]]`: 대문자 하나로 시작
- `[[:lower:]]+`: 소문자 1자 이상
- `$`: 끝

→ 첫 글자만 대문자, 나머지 소문자 → `Oracle` ✓

---

**[31번] 정답: ②**

이메일 유효성 검사의 충분한 패턴:
- 로컬 파트: `[[:alnum:]._-]+` (알파벳, 숫자, 점, 언더스코어, 하이픈)
- `@` 기호
- 도메인: `[[:alnum:].-]+`
- 점 + 최상위 도메인: `\.[[:alpha:]]{2,}`
- `'i'`: 대소문자 무관

①과 ③은 너무 단순, ④는 `@` 앞뒤에 임의 문자(`.`)로 너무 허용적.

---

**[32번] 정답: ②**

`[^[:digit:]]`: 숫자가 아닌 모든 문자 → `''`(빈 문자열)로 교체 → 삭제.
`'Phone: 010-1234-5678'` → 숫자 아닌 문자 제거 → `'01012345678'`

---

**[33번] 정답: ②**

REGEXP_INSTR `return_option`:
- `0` (기본): 패턴 일치 **시작** 위치 반환
- `1`: 패턴 일치 **끝 다음** 위치 반환 (패턴 이후를 처리할 때 유용)

---

**[34번] 정답: ④**

`[[:space:]]`는 공백 문자류: 스페이스, 탭(\t), 개행(\n), 폼피드(\f) 등. 숫자 0은 `[[:digit:]]`.

---

**[35번] 정답: ②**

CHECK 제약 조건에서 정규 표현식 사용:
```sql
CHECK (REGEXP_LIKE(column_name, 'pattern'))
```
REGEXP_LIKE만 TRUE/FALSE를 반환하므로 CHECK에 사용 가능.

---

**[36번] 정답: ②**

`'abcabc'`에서 `'a'` 발생: `a(1)bc a(4)bc` → 2회. `'i'`는 대소문자 무관이지만 여기서는 모두 소문자 `a`이므로 결과 동일.

---

**[37번] 정답: ②**

`^[0-9]{5}$`:
- `^`: 시작
- `[0-9]{5}`: 숫자 정확히 5자리
- `$`: 끝
③은 `^`와 `$`가 없어 더 긴 문자열의 일부와도 일치.
④는 `+`로 자리 수 제한 없음.

---

**[38번] 정답: ②**

`'2024-12-25'`에서 `[0-9]+`:
- 1번째: 2024
- 2번째: 12 ← `occurrence=2`
- 3번째: 25

---

**[39번] 정답: ①**

`REGEXP_COUNT(phone_number, '\.')`로 마침표 개수를 세어 2인 행 필터링. `'.'`(이스케이프 없음)는 임의 문자를 세어 잘못된 결과. ③은 INSTR로 접근 가능하지만 복잡.

---

**[40번] 정답: ①**

`([[:upper:]][[:lower:]]*)`: 대문자 하나 + 소문자 0개 이상 그룹 → `[\1]`로 교체.
- `Oracle` → `[Oracle]`
- `19c` → `19c` (숫자로 시작, 대문자로 시작 안 함)
- `Database` → `[Database]`

결과: `[Oracle] 19c [Database]`

## 상(심화) 모범답안

---

**[41번] 정답: ③**

`'one two three four'`에서 `[^ ]+`(공백 아닌 연속 문자):
- 1번째: one
- 2번째: two
- 3번째: three ← `occurrence=3`
- 4번째: four

---

**[42번] 정답: ②**

`([^ ]+) ([^ ]+)`: 공백 전 그룹(\1=John) 공백 후 그룹(\2=Smith).
- 6번째 인수 `1` → `\1` = John (이름)
- 6번째 인수 `2` → `\2` = Smith (성) ← 정답
④도 가능하지만 REGEXP_SUBSTR로 서브표현식 사용이 더 명확.

---

**[43번] 정답: ①**

`^[[:upper:]][[:alnum:]]{7}$`:
- `^[[:upper:]]`: 대문자 1자로 시작
- `[[:alnum:]]{7}`: 알파벳 또는 숫자 정확히 7자
- `$`: 끝
→ 총 8자, 첫 글자는 대문자

`Oracle19` = O(대문자) + racle19(7자) = 8자 → ✓
`ORACLE19` = O + RACLE19(7자) → 두 번째 이후 `[:alnum:]`이므로 ✓이지만, 첫 글자가 `[[:upper:]]`이고 나머지가 `[[:alnum:]]`이면 둘 다 가능.
실제로는 ①이 더 자연스러운 정답 (대문자 시작 + 혼합).

---

**[44번] 정답: ④**

세 가지 방법 모두 "숫자 없는 이메일" 조회:
- ①: `NOT REGEXP_LIKE` → 직접 부정
- ②: `^[^[:digit:]]+$` → 처음~끝까지 비숫자
- ③: `REGEXP_COUNT = 0` → 숫자 발생 횟수 0

모두 동일한 결과를 반환한다.

---

**[45번] 정답: ②**

`<[^>]+>`: `<` 뒤에 `>`가 아닌 문자 1개 이상 + `>`.
①의 `<.*>`는 greedy(최대 일치)라서 `<b>text</b>` 전체가 하나의 태그로 처리될 수 있음.
②의 `[^>]+`는 `>`를 넘지 않아 각 태그를 정확히 제거.
③은 Oracle 정규 표현식이 `.*?` (비탐욕) 완전 지원이 제한적.

---

**[46번] 정답: ②**

`'abcabc'`에서 `'b'`:
- 1번째 발생: 위치 2
- 2번째 발생: 위치 5 ← `occurrence=2`

---

**[47번] 정답: ③**

②: `^[0-9]{4}(-[0-9]{4}){3}$` → XXXX-XXXX-XXXX-XXXX ✓  
③: `^[0-9]{4}-[0-9]{4}-[0-9]{4}-[0-9]{4}$` → XXXX-XXXX-XXXX-XXXX ✓

두 패턴 모두 XXXX-XXXX-XXXX-XXXX 형식에 일치 → ④가 정답이지만,
④를 선택지로 제공했다면 ②와 ③ 모두 올바르다.

답: ③ (명시적 패턴이 더 가독성 좋음)

---

**[48번] 정답: ②**

REGEXP_REPLACE에서 `\1`, `\2`, ... = **역참조(Back Reference)**:
- 패턴의 `()` 그룹으로 캡처한 문자열을 replace_string에서 재사용
- `\1` = 첫 번째 괄호 그룹에 일치한 문자열
- `\2` = 두 번째 괄호 그룹 ... 등

```sql
REGEXP_REPLACE('2024-12', '([0-9]{4})-([0-9]{2})', '\2/\1')
-- 결과: 12/2024  (연월 순서 바꾸기)
```

**[49번] 정답: ②**

`'the cat sat on the mat'`에서 모음(`[aeiou]`) 개수:
- t**h**e(e=1) c**a**t(a=2) s**a**t(a=3) **o**n(o=4) th**e**(e=5) m**a**t(a=6)
→ 6개

**[50번] 정답: ④**

세 패턴 모두 "대문자 2자 + 언더스코어 + 대문자 3자" 형식에 일치:
- `[A-Z]{2}` = `[[:upper:]]{2}` = `[A-Z][A-Z]` → 동일
- `_` = 언더스코어 리터럴
- `[A-Z]{3}` = `[[:upper:]]{3}` = `[A-Z][A-Z][A-Z]` → 동일

HR 스키마 예: `IT_PROG`, `SA_REP`, `AD_PRES` 등 모두 일치.
