## 하(기초) 모범답안

| 번호 | 정답 | 해설 |
|------|------|------|
| 1 | ③ | Oracle 10g에서 POSIX 정규표현식 지원 도입 |
| 2 | ③ | `.`은 임의의 한 문자(줄바꿈 제외), `%`와 `_`는 LIKE 연산자의 와일드카드 |
| 3 | ② | `^`는 문자열 시작 앵커, `$`는 끝 앵커 |
| 4 | ③ | `+`는 1회 이상, `*`는 0회 이상, `?`는 0회 또는 1회 |
| 5 | ② | `[aeiou]`는 문자 집합 — 나열된 문자 중 하나와 일치 |
| 6 | ③ | REGEXP_LIKE는 조건식으로 TRUE/FALSE 반환, WHERE 절에서 사용 |
| 7 | ③ | REGEXP_INSTR는 일치 없을 때 0 반환 (INSTR와 동일) |
| 8 | ④ | REGEXP_SUBSTR는 일치 없을 때 NULL 반환 |
| 9 | ② | `i` = case-insensitive(대소문자 무시), `c` = case-sensitive(기본값) |
| 10 | ③ | `[:digit:]`는 `[0-9]`와 동일한 POSIX 숫자 클래스 |
| 11 | ② | `{n}`은 정확히 n회 반복, `{n,}`은 n회 이상, `{n,m}`은 n~m회 |
| 12 | ③ | Oracle 정규표현식에서 역참조는 `\1`, `\2` 사용 (PCRE의 $1과 다름) |
| 13 | ③ | REGEXP_COUNT는 Oracle 11g에서 추가 |
| 14 | ② | `[^0-9]`는 부정 문자 집합 — 0~9 숫자가 아닌 한 문자 |
| 15 | ③ | REGEXP_LIKE는 WHERE 절 조건식으로 사용 (HAVING도 가능) |
| 16 | ① | `^[A-Z]`는 대문자로 시작하는 문자열과 일치 |
| 17 | ③ | `?`는 앞 문자(u) 0회 또는 1회 → "color"와 "colour" 모두 일치 |
| 18 | ② | return_opt=1이면 일치 끝 위치 +1 반환 (다음 시작 위치) |
| 19 | ② | `.+` = 임의 문자 1회 이상이므로 최소 1글자 |
| 20 | ① | 다중행 모드('m')에서 `^`/`$`는 문자열 전체가 아닌 각 줄의 시작/끝에 적용 |

## 중(응용) 모범답안

---

**[21번] 정답: ②**

`^[AEIOU]` 패턴: `^`는 문자열 시작, `[AEIOU]`는 모음 중 하나. `'i'` 옵션으로 소문자 모음도 포함.
→ first_name의 첫 글자가 모음인 직원을 반환.

---

**[22번] 정답: ③**

`'515.123.4567'`에서 `[0-9]+` 패턴의 두 번째(`occurrence=2`) 일치:
- 첫 번째: `515` (위치 1)
- 두 번째: `123` (위치 **5**)

정답: **5**

---

**[23번] 정답: ②**

`'Oracle Database 19c'`에서 `[A-Za-z]+` 패턴의 두 번째(`occurrence=2`) 일치:
- 첫 번째: `Oracle`
- 두 번째: **`Database`**

`19c`는 `[A-Za-z]+` 패턴과 숫자가 섞여 있어 일치하지 않음 (숫자 19 부분 제외 후 `c`가 세 번째).

---

**[24번] 정답: ①**

`\s+`는 공백 문자 1개 이상. `REGEXP_REPLACE`의 `occurrence` 기본값은 0(모든 일치 치환). 'Hello   World'의 3개 연속 공백 → 단일 공백으로 치환 = `'Hello World'`.

---

**[25번] 정답: ②**

`'Mississippi'`에서 `'s'` (대소문자 무시) 출현 횟수:
M-i-**s**-**s**-i-**s**-**s**-i-p-p-i → **4**회

---

**[26번] 정답: ②**

`^[A-Z]{4,6}$`: 대문자([A-Z]) 4~6자로만 구성된 문자열.
HR 스키마의 email 컬럼은 대문자 알파벳만으로 구성 → `'i'` 옵션 없이도 일치.

---

**[27번] 정답: ②**

`^(\S+)\s+(\S+)$` — 두 그룹 캡처:
- `\1` = 'King' (첫 번째 비공백 그룹)
- `\2` = 'Steven' (두 번째 비공백 그룹)

교체 패턴 `'\2 \1'` → **'Steven King'**

---

**[28번] 정답: ②**

`\d{3}\.\d{3}\.\d{4}`:
- `\d{3}` = 숫자 3자리
- `\.` = 점(.) 한 개
- `\d{4}` = 숫자 4자리

→ `'123.456.7890'`이 유일하게 일치.

---

**[29번] 정답: ①**

`'[A-Z'`는 `]`로 닫히지 않은 잘못된 정규표현식 패턴 → Oracle이 `ORA-12726: unterminated bracket expression in regular expression` 오류 발생.

---

**[30번] 정답: ②**

`'abc123def456'`에서 `[0-9]+` 첫 번째(`occurrence=1`) 일치 → **'123'**

---

**[31번] 정답: ①**

`(.)`는 임의의 한 문자를 그룹 캡처, `\1`은 역참조(같은 문자). `(.)\1`은 같은 문자가 **연속으로 2회** 반복되는 패턴.
예: "Nneena" → 'nn', "Higgins" → 'gg'

---

**[32번] 정답: ③**

`'abcabcabc'`에서 `'abc'` 세 번째(`occurrence=3`) 일치의 시작 위치:
- 1번째: 위치 1
- 2번째: 위치 4
- 3번째: 위치 **7**

---

**[33번] 정답: ①**

`REGEXP_REPLACE(str, '[0-9]', '')`: 숫자 한 자리를 빈 문자열로 치환 (전체 반복 = 모든 숫자 제거).
② `[^0-9]`는 숫자 아닌 문자를 제거. ③ replace_string에 NULL은 오류 가능성. ④ 'X'로 치환.

---

**[34번] 정답: ②**

`'aababab'`에서 `'ab'` 패턴 출현:
- a**ab**abab → 위치 2
- aa**b**... 아니고: a-a-b-a-b-a-b
  - 위치 2: `ab`
  - 위치 4: `ab`
  - 위치 6: `ab`
→ **3**회

※ `REGEXP_COUNT`는 겹치지 않는 일치를 기준으로 카운트.

**[35번] 정답: ②**

`(\d{4})-(\d{2})-(\d{2})` 패턴에서:
- 그룹 1(`subexpr=1`): '2024'
- 그룹 2(`subexpr=2`): **'01'**
- 그룹 3(`subexpr=3`): '15'

`subexpr=2`이므로 **'01'** 반환.

**[36번] 정답: ③**

Oracle 정규표현식의 `match_param` 기본값은 **'c'(대소문자 구분)**. 'i'는 대소문자 무시이며 Oracle 버전 제한 없음(10g+). 두 옵션 동시 지정은 오류.

## 상(심화) 모범답안

---

**[37번] 정답: ①**

`([aeiou])` — 모음을 캡처 그룹으로 지정, 역참조 `\1`로 동일 모음을 대괄호로 감쌈.

```sql
SELECT last_name,
       REGEXP_REPLACE(last_name, '([aeiou])', '[\1]', 1, 0, 'i') AS masked
FROM   employees
WHERE  employee_id IN (100, 101, 102);
```

| last_name | masked |
|-----------|--------|
| King | K[i]ng |
| Kochhar | K[o]chh[a]r |
| De Haan | D[e] H[a][a]n |

---

**[38번] 정답: ①**

`^[^AEIOU][AEIOU]`:
- `^` : 시작
- `[^AEIOU]` : 모음이 아닌 문자 (자음) 한 개
- `[AEIOU]` : 모음 한 개

→ 첫 글자 자음, 두 번째 글자 모음인 last_name.
예: King(`Ki`), Kochhar(`Ko`), De Haan(`De`)

---

**[39번] 정답: ②**

`subexpr=3`은 전체 패턴 내 **세 번째 캡처 그룹** `(\d+)`의 내용을 추출.

- 그룹 1: `(\w+)` = 'John'
- 그룹 2: `(\w+)` = 'Smith'
- 그룹 3: `(\d+)` = **'30'**

`subexpr=0`이면 전체 일치 'John Smith 30' 반환.

---

**[40번] 정답: ②**

`'Hello World'`에서 `'o'` **두 번째** 일치 위치:
- 첫 번째 'o': 위치 5 (Hell**o**)
- 두 번째 'o': 위치 **8** (W**o**rld)

**[41번] 정답: ②**

`^(\d{3})\.(\d{3})\.(\d{4})$`:
- 그룹1: 지역번호(515), 그룹2: 국번(123), 그룹3: 번호(4567)

교체 패턴 `'(\1) \2-\3'` → `'(515) 123-4567'`

**[42번] 정답: ①**

```
'^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$'
```

- `[A-Za-z0-9._%+-]+` : 로컬파트 (1회 이상)
- `@` : @ 기호
- `[A-Za-z0-9.-]+` : 도메인
- `\.` : 점
- `[A-Za-z]{2,}` : TLD 2자 이상

② 너무 제한적(알파벳만), ③ LIKE 패턴(정규식 아님), ④ 너무 느슨.

---

**[43번] 정답: ①**

`'the cat sat on the mat'`에서 `at`으로 끝나는 단어:
- c**at** — 일치
- s**at** — 일치
- m**at** — 일치

`REGEXP_COUNT`는 겹치지 않는 `at` 출현이 아닌 단어 경계 기준으로 **3**회.

---

**[44번] 정답: ①**

`CONNECT BY LEVEL`로 `LEVEL`을 1부터 패턴 출현 횟수까지 증가시키며 `REGEXP_SUBSTR(..., LEVEL)`로 각 occurrence를 추출. `LISTAGG`로 콤마 연결.
→ 쉼표 없는 구분자 문자열에서 토큰을 모두 추출하여 연결.

---

**[45번] 정답: ③**

`occurrence=0` (기본값) → 패턴과 일치하는 **모든** 부분을 치환.
`occurrence=n` (n≥1) → n번째 일치만 치환.

---

**[46번] 정답: ①**

`^[A-Z][a-z]+$`:
- `^` : 시작
- `[A-Z]` : 첫 글자 대문자 1개
- `[a-z]+` : 이후 소문자 1개 이상
- `$` : 끝

→ 첫 글자만 대문자, 나머지 전체 소문자인 last_name.

---

**[47번] 정답: ①**

- 쿼리 A: `'king'` 패턴이 어디든 포함되면 일치 → 'Kingsleigh', 'Yoking' 등도 해당
- 쿼리 B: `'^king$'` — 시작부터 끝까지 정확히 'king'(대소문자 무시)인 행만 → 'King' 1명

```sql
-- A: 'king' 포함
WHERE REGEXP_LIKE(last_name, 'king', 'i')      -- 더 많은 행

-- B: 정확히 'king'
WHERE REGEXP_LIKE(last_name, '^king$', 'i')    -- King 1명
```

**[48번] 정답: ②**

```sql
LENGTH(last_name)
- LENGTH(REGEXP_REPLACE(last_name, '[aeiou]', '', 1, 0, 'i'))
```

- `REGEXP_REPLACE(... '[aeiou]', '')` : 모음 제거 후 문자열 길이
- 원래 길이 - 모음 제거 후 길이 = **모음의 수**

`REGEXP_COUNT(last_name, '[aeiou]', 1, 'i')`와 동일한 결과를 LENGTH 차이로 계산하는 방법.

---

**[49번] 정답: ①**

**정규표현식은 인덱스를 활용하지 못한다.** `REGEXP_LIKE`는 함수 기반 조건으로 B-Tree 인덱스 스캔이 불가하여 Full Table Scan 유발. 함수 기반 인덱스(FBI)를 별도 생성해야 인덱스 활용 가능.

나머지는 모두 옳은 설명:
- ② 복잡한 패턴 = 더 많은 CPU
- ③ 단순 패턴 = LIKE가 유리
- ④ 정규식 = FTS 가능성

---

**[50번] 정답: ①**

CSV 문자열 파싱 패턴:

```sql
SELECT TRIM(REGEXP_SUBSTR('a,bb,ccc,dddd', '[^,]+', 1, LEVEL)) AS token
FROM   dual
CONNECT BY LEVEL <= REGEXP_COUNT('a,bb,ccc,dddd', ',') + 1;
```

- `[^,]+` : 콤마가 아닌 문자 1개 이상 (각 토큰)
- `LEVEL` : 1, 2, 3, 4로 증가 → occurrence로 사용
- `REGEXP_COUNT(..., ',') + 1` : 콤마 개수 + 1 = 토큰 개수(4)

결과:
```
TOKEN
-----
a
bb
ccc
dddd
```
